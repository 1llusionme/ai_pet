# MindShadow 公开发布清单（当前轮次）

## 目标
- 让产品从“可自己用/可小范围分享”推进到“可公开发布”的最小闭环。
- 本轮只做高价值、可落地、可验证的发布能力，不做大而全改造。

## 本轮清单
- [x] 统一 API 请求链路标识与错误返回，提升线上排障效率。
- [x] 增加发布就绪检查接口，集中暴露关键依赖健康状态。
- [x] 增加受保护的运维指标接口，支持基础运行监控。
- [x] 完善前端服务状态提示，明确“在线 / 降级 / 需配置密钥”。
- [x] 补齐自动化门禁（后端测试 + 前端构建）并可重复执行。

## 验收标准
- [x] 所有 API 响应带 `X-Request-ID`。
- [x] 非预期异常返回统一 JSON，包含 `request_id`。
- [x] `GET /api/ready` 可返回 db、kb、uploads、scheduler 的状态。
- [x] `GET /api/ops/metrics` 在配置运维令牌后才可访问。
- [x] 前端顶部状态文案可区分在线、离线、模型未配置。
- [x] 自动化流程可执行并通过（本地或 CI 环境至少一处验证）。
