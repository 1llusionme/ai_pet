# iOS TestFlight 与上架执行清单

## A. 账号与证书

- [ ] Apple Developer 账号可用且已缴费。
- [ ] App Store Connect 已创建 App（Bundle ID 一致）。
- [ ] 证书与描述文件可用于真机和发布构建。
- [ ] 团队权限已分配（开发、发布、财务至少 1 人可操作）。

## B. 构建与环境

- [ ] iOS 构建配置区分 dev/staging/prod。
- [ ] 生产 API 域名为 HTTPS，证书有效。
- [ ] 服务端 `/api/health` 可稳定访问。
- [ ] 服务端模型配置已完成，异常时有兜底返回。
- [ ] 崩溃与错误日志可被收集。

## C. 聊天主链路验收（原生）

- [ ] 首屏可进入聊天页，界面无布局错位。
- [ ] 消息发送成功，AI 回复正常显示。
- [ ] 流式输出期间加载态稳定。
- [ ] 网络异常时有明确提示且可重试。
- [ ] 会话切换可用，新对话不展示旧消息。
- [ ] 模型不可用时状态提示清晰。

## D. 混合能力验收（WebView）

- [ ] 非核心页面可从原生入口正常打开。
- [ ] WebView 返回逻辑符合 iOS 习惯。
- [ ] Web 页和原生页切换无明显卡顿或白屏。

## E. TestFlight 提交前

- [ ] 版本号与 Build 号已递增。
- [ ] App 名称、图标、启动图符合规范。
- [ ] 测试账号与测试说明已准备。
- [ ] Release Notes 已写明本次改动和已知问题。

## F. App Store 提审素材

- [ ] 应用描述（简短描述 + 详细描述）已完成。
- [ ] 6.7"/6.5"/5.5" 截图准备完成。
- [ ] 隐私政策 URL 与服务条款 URL 可公开访问。
- [ ] 支持页面 URL 可访问。
- [ ] 审核备注写明混合方案与原生聊天主链路价值。

## G. 隐私与合规

- [ ] Data Collection 问卷与真实行为一致。
- [ ] 权限申请文案（通知、相册/文件）清晰且必要。
- [ ] 用户数据删除/导出路径已定义。
- [ ] AI 内容风险策略可说明（免责声明或反馈入口）。

## H. 发布后 48 小时

- [ ] 核心指标看板可查看（成功率、崩溃率、平均响应时长）。
- [ ] P0 问题响应人和处理时限明确。
- [ ] 回滚版本可随时重新发布。
