# 方案A：不依赖Xcode的手机可演示上线教程

## 1. 目标与结果

- 目标：在不打 iOS 安装包的情况下，让你在面试现场用手机直接演示「上岸！」。
- 最终结果：
  - 一个前端线上地址（手机浏览器可打开）。
  - 一个后端线上地址（前端可正常请求）。
  - 手机可通过“添加到主屏幕”像 App 一样打开。

## 2. 路线选择

- 前端：Vercel（部署 `docs/idea2mvp/年轻人风格原型设计`）
- 后端：Render（部署 `server`）
- 原因：新手操作路径最短、图形化界面友好、免费额度足够做面试演示。

## 3. 上线前准备（10 分钟）

- 注册账号：
  - GitHub
  - Render
  - Vercel
- 本地确认项目可跑（你已经完成大部分）：
  - 前端：`npm run dev`
  - 后端：`python -m server.app` 或 `./.venv-py313/bin/python -m server.app`
- 准备一个可用的大模型 API Key（例如 OpenAI 兼容服务）。

## 4. 第一步：把代码推到 GitHub

1. 打开终端进入项目根目录：

```bash
cd /Users/liyulin/ai_pet
```

2. 确认当前分支代码完整后推送到你的 GitHub 仓库。
3. 确认仓库里存在两个关键目录：
   - `server`
   - `docs/idea2mvp/年轻人风格原型设计`

## 5. 第二步：部署后端到 Render

### 5.1 创建服务

1. 打开 Render 控制台，点击 `New +` -> `Web Service`。
2. 选择你的 GitHub 仓库。
3. 填写：
   - Name：`shang-an-server`
   - Root Directory：`server`
   - Runtime：`Python 3`
   - Build Command：`pip install -r requirements-prod.txt`
   - Start Command：`gunicorn -c gunicorn.conf.py server.wsgi:app`

### 5.2 配置环境变量

在 Render 的 Environment 里新增以下变量：

- `MINDSHADOW_HOST=0.0.0.0`
- `MINDSHADOW_PORT=10000`
- `MINDSHADOW_DB_PATH=/opt/render/project/src/data/mindshadow.db`
- `MINDSHADOW_UPLOAD_DIR=/opt/render/project/src/data/uploads`
- `MINDSHADOW_KB_PATH=/opt/render/project/src/server/knowledge_base/teaching_exam_kb.json`
- `MINDSHADOW_LLM_PROVIDER=openai_compatible`
- `MINDSHADOW_LLM_BASE_URL=https://api.openai.com/v1/chat/completions`
- `MINDSHADOW_LLM_MODEL=gpt-4o-mini`
- `MINDSHADOW_LLM_API_KEY=你的真实Key`
- `MINDSHADOW_OPS_TOKEN=你自定义一串长随机字符串`
- `MINDSHADOW_MEMORY_DEBUG_ENABLED=0`
- `MINDSHADOW_HYBRID_RECALL_ENABLED=1`
- `MINDSHADOW_ENABLE_WEB_SEARCH=true`

### 5.3 验收后端

部署成功后会得到一个地址，例如：

- `https://shang-an-server.onrender.com`

在浏览器访问：

- `https://shang-an-server.onrender.com/api/health`

看到 `ok` 字段即为成功。

## 6. 第三步：部署前端到 Vercel

### 6.1 创建项目

1. 打开 Vercel 控制台，点击 `Add New` -> `Project`。
2. 选择同一个 GitHub 仓库。
3. 关键配置：
   - Framework Preset：`Vite`
   - Root Directory：`docs/idea2mvp/年轻人风格原型设计`
   - Build Command：`npm run build`
   - Output Directory：`dist`

### 6.2 配置前端环境变量

在 Vercel 项目设置中添加：

- `VITE_API_BASE_URL=https://你的Render后端域名`
- `VITE_ROUTER_BASENAME=/`

保存后重新部署。

### 6.3 验收前端

部署成功后会得到前端地址，例如：

- `https://shang-an-web.vercel.app`

手机打开后应能进入聊天页面并正常收发消息。

## 7. 第四步：手机“类 App”展示

1. iPhone 用 Safari 打开你的前端地址。
2. 点击分享按钮 -> `添加到主屏幕`。
3. 名称填：`上岸！`。
4. 回到桌面点击图标即可全屏体验，适合面试现场展示。

## 8. 第五步：面试兜底方案

- 兜底 1：录制一段 2 分钟完整演示视频（主页 -> 提问 -> AI 回复 -> 历史消息）。
- 兜底 2：准备一页架构图（前端 Vercel、后端 Render、LLM API）。
- 兜底 3：准备“已知限制”话术：
  - 当前为 Web 优先上线，保障可用性与迭代速度。
  - 后续按路线图逐步原生化 iOS 主链路。

## 9. 常见问题与快速修复

- 前端能开但不能聊天：
  - 检查 `VITE_API_BASE_URL` 是否是正确的 Render 域名。
  - 检查 Render 的后端服务状态是否为 `Live`。
- 后端 500：
  - 优先检查 `MINDSHADOW_LLM_API_KEY` 是否有效。
  - 检查 `MINDSHADOW_LLM_BASE_URL` 与模型名是否匹配。
- 跨域报错：
  - 当前后端默认允许跨域，若仍报错，先确认前端是否请求到了正确域名。

## 10. 你的执行顺序（直接照做）

1. 推代码到 GitHub。
2. 在 Render 创建后端服务并配好环境变量。
3. 用 `/api/health` 验收后端。
4. 在 Vercel 创建前端项目并配置 `VITE_API_BASE_URL`。
5. 手机打开前端地址验证聊天。
6. 添加到主屏幕并录制演示视频作为兜底。
