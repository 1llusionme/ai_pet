# MindShadow iOS 混合方案执行规划（原生化聊天主链路）

## 1. 目标

- 2 周内进入 TestFlight 内测。
- 保留现有 Web 功能可用的前提下，把聊天主链路做成原生体验。
- 审核通过率优先，避免“纯网页套壳”风险。

## 2. 版本策略

### V1（TestFlight 首包）

- iOS 容器承载现有 Web 功能页。
- 聊天主链路原生化：输入框、消息列表、加载态、错误态、会话切换。
- 后端 API 复用现有接口：`/api/chat/stream`、`/api/history`、`/api/health`、上传接口。

### V1.1（提审包）

- 补齐原生状态管理与弱网恢复。
- 完成隐私与审核素材。
- 完成提审前全量验收。

### V2（上架后迭代）

- 学习资产、设置页逐步原生化。
- 降低 WebView 依赖，提升性能与一致性。

## 3. 范围定义

### 本轮必须交付

- 原生聊天页（消息流、输入、发送、打字中、失败重试）。
- 原生会话切换能力（会话列表、创建新对话、切换历史）。
- iOS 壳容器与 Web 功能页互通。
- 生产部署配置、日志与健康检查。
- TestFlight 执行清单与提审清单。

### 本轮不做

- A/B 评测台原生化。
- 全量运营后台原生化。
- 复杂离线缓存策略。

## 4. 架构方案

- 客户端：SwiftUI + URLSession（原生聊天）+ WKWebView（承载非核心功能）。
- 服务端：Flask API 保持不拆分，统一域名。
- 鉴权：MVP 阶段延续现有 user_id 机制，后续接账号体系。
- 观测：健康检查 + 接口错误日志 + 客户端关键埋点。

## 5. 执行阶段与里程碑

### 阶段 A：环境与部署基线（Day 1-2）

- 产出生产配置模板（环境变量、启动命令、Gunicorn）。
- 固化后端部署说明和回滚策略。
- 里程碑：后端可稳定生产启动，健康检查可用。

### 阶段 B：原生聊天主链路（Day 3-7）

- 完成聊天页 UI 与交互。
- 对接历史加载与流式回复。
- 完成发送失败、超时、弱网提示与重试。
- 里程碑：原生聊天完成端到端可用。

### 阶段 C：混合集成（Day 8-10）

- 保留 WebView 页承载非核心功能。
- 原生页与 Web 页入口统一。
- 里程碑：功能闭环，用户可完整使用。

### 阶段 D：提审准备（Day 11-14）

- TestFlight 内测、问题修复。
- 完成 App Store 文案、截图、隐私声明。
- 里程碑：具备提审条件。

## 6. 验收标准

- 消息发送成功率 >= 99%（内测样本）。
- API 健康检查稳定可达。
- 首屏交互时间 <= 2.5 秒（常规网络）。
- 聊天链路故障可恢复（重试或降级提示）。

## 7. 风险与预案

- 审核认为套壳：聊天主链路必须原生，补充审核说明。
- 模型波动：服务端返回兜底文案并标识状态。
- 弱网断流：客户端支持超时提示与重新发送。

## 8. 本周执行清单

1. 完成后端生产部署资产与说明文档。
2. 完成 iOS TestFlight 与提审清单文档。
3. 开始原生聊天模块工程骨架。
4. 完成第一轮端到端联调。
