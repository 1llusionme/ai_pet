# MindShadow 新增需求文档：引用来源句（RAG 透明化）

**版本**: 0.9（新增需求草案）  
**状态**: 待评审  
**提出背景**: 当前回答可能出现“检索到多源信息后口径混合”，用户难以判断结论依据。  
**目标版本**: 当前 Web 主界面增量迭代（不新增页面）  
**负责人**: 产品 + 前端 + 后端

---

## 1. 需求背景与问题定义

### 1.1 现状问题
用户在学习问答中经常只能看到“结论”，但看不到该结论具体来自哪一句资料，导致：
1. 用户无法快速核验答案是否与自己教材一致。  
2. 当不同资料口径不一致时，用户不知道模型依据了哪一条。  
3. 对 RAG 的信任建立慢，容易把“模型生成错误”误判为“系统没检索到”。  

### 1.2 本次要解决的问题
在不改变当前产品“对话优先”体验的前提下，新增“引用来源句”能力，让用户能在聊天流中一键查看“这句话从哪来”。

### 1.3 非目标（本期不做）
1. 不做“考试口径优先”强约束。  
2. 不做复杂文献管理页面。  
3. 不做用户手工上传来源的逐句标注工具。  
4. 不重构现有聊天主路径与视觉风格。

---

## 2. 目标用户与价值

### 2.1 目标用户
1. 备考用户（教招/教师资格相关）  
2. 对答案准确性敏感、需要“可核验依据”的高频学习者  

### 2.2 用户价值
1. **可核验**：回答能追溯到具体来源句，减少“盲信/盲疑”。  
2. **可学习**：用户可直接复读来源句，提升记忆效率。  
3. **可决策**：出现口径冲突时，用户能快速判断要采用哪套教材口径。  

---

## 3. 功能范围

### 3.1 功能名称
“引用来源句” （Source Quote）

### 3.2 功能描述
当 AI 回答使用到知识库、语义记忆或联网信息时，系统为该条 AI 消息挂载可展开的“来源句卡片”，包含：
1. 来源类型（知识库/记忆/联网）  
2. 来源标题（如《教师招聘重点背诵知识点》）  
3. 页码或链接（若有）  
4. 命中的原句片段（source quote）  
5. 与回答的对应关系（可选短标签：如“支撑结论”“补充背景”）

### 3.3 展示触发条件
1. 当本次回答存在 `kb_sources`、`semantic_sources`、`sources` 任一非空时展示。  
2. 三者均为空时不展示来源入口。  
3. 在流式回复中，来源入口仅在 `done` 事件后出现，避免抖动。

## 4. 交互设计（严格基于当前 UI）

## 4.1 设计原则
1. 不新增新页面，全部在现有聊天页完成。  
2. 不破坏当前“用户气泡/AI气泡/系统气泡”结构。  
3. 维持当前视觉语义：圆角卡片、柔和渐变、轻量动效。  

### 4.2 页面锚点与改动范围
1. **聊天主页面容器**：沿用 [ChatPage.tsx](file:///Users/liyulin/ai_pet/docs/idea2mvp/年轻人风格原型设计/src/app/pages/ChatPage.tsx#L699-L758) 的消息流。  
2. **消息气泡组件**：在 [MessageBubble.tsx](file:///Users/liyulin/ai_pet/docs/idea2mvp/年轻人风格原型设计/src/app/components/ui/MessageBubble.tsx#L24-L121) 的 AI 气泡下方新增“来源入口条”。  
3. **输入区不新增主按钮**：保持 [InputArea.tsx](file:///Users/liyulin/ai_pet/docs/idea2mvp/年轻人风格原型设计/src/app/components/ui/InputArea.tsx#L42-L95) 现有交互负担。

### 4.3 UI 呈现方案（推荐）
1. **AI 气泡下方增加轻量入口**  
   - 文案：`查看引用来源（2）`  
   - 样式：胶囊按钮，和系统提示同色系（蓝紫浅底）  
   - 仅对含来源的 AI 消息显示  
2. **点击后在该气泡下原位展开“来源卡片组”**  
   - 每条卡片包含：来源标签、标题、页码/域名、引用原句  
   - 默认展示最多 2 条，支持“展开更多”  
3. **来源句交互**  
   - 支持一键复制原句  
   - 支持“跳转原文”（网页来源跳链接；本地知识显示标题+页码）  

### 4.4 交互流程
1. 用户提问 -> AI 回复完成。  
2. 若有来源，气泡下显示“查看引用来源（N）”。  
3. 点击后展开来源卡片。  
4. 用户可：
   - 复制引用句；  
   - 打开网页来源；  
   - 收起来源卡片。  
5. 切换消息不影响其他消息的展开状态（每条消息独立控制）。

### 4.5 异常与边界体验
1. **有来源但缺页码**：显示“章节定位”，不显示空页码。  
2. **来源句过长**：两行截断 + “展开全文”。  
3. **链接失效**：toast 提示“来源链接暂不可访问”。  
4. **仅语义记忆来源**：明确标签“历史记忆引用”，避免误解为教材原文。

## 5. 信息架构与数据结构（产品视角）

### 5.1 前端消息模型新增字段
在现有 `Message` 基础上增加：
1. `citations?: CitationItem[]`  
2. `citationSummary?: { total: number; hasKb: boolean; hasSemantic: boolean; hasWeb: boolean }`  

### 5.2 CitationItem 建议结构
1. `sourceType`: `"kb" | "semantic" | "web"`  
2. `title`: string  
3. `page?`: string  
4. `url?`: string  
5. `quote`: string  
6. `confidence?`: number（预留，不强依赖）  

### 5.3 后端响应兼容策略
1. 保持原字段不删除（`kb_sources/semantic_sources/sources`）。  
2. 新增聚合字段 `citations`，方便前端一次渲染。  
3. 若暂无 `citations`，前端可用旧字段本地映射，保障灰度兼容。

---

## 6. 业务规则

1. 同一条回答最多展示 5 条引用来源，默认展开 2 条。  
2. 至少保留 1 条“支撑主结论”来源。  
3. 当来源之间存在明显冲突时，来源卡片顶部提示：`检测到多来源口径，建议按你的教材口径为准`。  
4. 不允许把“模型生成句”当作来源句展示。来源句必须来自检索上下文原文片段。

## 7. 埋点与效果评估

### 7.1 关键埋点
1. `citation_entry_exposed`（来源入口曝光）  
2. `citation_entry_clicked`（来源入口点击）  
3. `citation_copy_clicked`（复制来源句）  
4. `citation_open_source_clicked`（打开来源）  
5. `citation_conflict_banner_exposed`（冲突提示曝光）

### 7.2 核心指标
1. 来源入口点击率（CTR）  
2. 引用句复制率  
3. “答案不可信”负反馈率（目标下降）  
4. 二次追问率中“求依据”类占比（目标下降）

## 8. 验收标准（DoD）

### 8.1 功能验收
1. 含来源的 AI 消息均能显示“查看引用来源（N）”。  
2. 点击后可稳定展开/收起来源卡片，不影响消息流滚动。  
3. 来源类型、标题、页码/链接、引用句显示完整。  
4. 无来源时不出现入口，不占版面。  

### 8.2 体验验收
1. 单手点击目标 >= 44px。  
2. 展开/收起动效在 300ms 内，不卡顿。  
3. 长引用场景下排版不破坏当前气泡布局。  

### 8.3 数据验收
1. 埋点完整上报。  
2. `citations` 字段缺失时仍可回退旧字段渲染。

## 9. 迭代计划建议

### Phase A（本期）
1. 聊天消息级来源入口 + 原位展开卡片。  
2. 复制引用句、打开来源链接。  
3. 基础埋点与监测看板。

### Phase B（后续可选）
1. 句级高亮“回答片段 ↔ 来源句”映射。  
2. 按“教材/记忆/联网”切换过滤来源。  
3. 用户侧“偏好来源”设置（仅展示教材、隐藏联网等）。

---

## 10. 风险与应对

1. **风险：来源过多导致阅读负担增加**  
   - 应对：默认折叠 + 前两条优先展示。  
2. **风险：来源冲突造成困惑**  
   - 应对：冲突提示文案 + 清晰来源类型标签。  
3. **风险：前后端字段升级不同步**  
   - 应对：兼容旧字段本地映射 + 灰度开关。
